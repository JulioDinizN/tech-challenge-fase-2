Oracle Cloud Architecture

The original challenge examples map to Oracle Cloud Infrastructure services. The final deployment targets OKE for Kubernetes workloads, OCIR for images, PostgreSQL resources for relational service data, Redis for evaluation cache, OCI Queue for asynchronous analytics events, and OCI NoSQL Database for analytics persistence.

Challenge category Oracle Cloud target Purpose
Kubernetes cluster OKE Runs the five microservice deployments, services, ingress, and autoscaling resources.
Container registry OCIR Stores versioned images built from the repository Dockerfiles.
Relational database PostgreSQL resources Stores auth API keys, feature flags, and targeting rules in separate service databases.
Cache OCI Cache with Redis or approved Redis deployment Supports low-latency evaluation reads and repeated targeting decisions.
Queue OCI Queue Decouples evaluation event production from analytics processing.
NoSQL database OCI NoSQL Database Persists analytics events consumed from the queue.

Service Responsibilities

Service Runtime Responsibility Data dependency
auth-service Go Issues and validates API keys used by protected services. PostgreSQL
flag-service Python Creates, lists, and retrieves feature flags. PostgreSQL and auth-service
targeting-service Python Creates and evaluates targeting rules. PostgreSQL and auth-service
evaluation-service Go Combines flag state, targeting decisions, Redis cache, and event production. Redis, flag-service, targeting-service, OCI Queue
analytics-service Python Consumes evaluation events and persists analytics data. OCI Queue and OCI NoSQL Database

Local Execution

Local execution uses Docker Compose to prove the ecosystem can run before cloud deployment. The stack starts five app containers and four dependency containers: two PostgreSQL instances, Redis, and DynamoDB Local for local analytics compatibility.

This satisfies the first challenge requirement by showing that each service can be built, configured, started, and exercised outside the cloud environment before the Kubernetes deployment is attempted.
